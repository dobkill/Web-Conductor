# 功能需求

## 一、DOM 上传

### 目标

插件需要能够将当前页面的结构化 DOM 快照发送给服务端。

### 需求说明

- 采集当前活动标签页的页面信息。
- 采集页面标题、URL、视口信息、滚动位置。
- 采集页面主要 DOM 结构。
- 为关键元素生成可定位信息，如 `xpath`、辅助 `selector`、标签名、文本摘要、属性摘要。
- 在发送前执行脱敏和裁剪。

### 其他上传内容

- `cookie`、`token`、`authorization` 相关值


### 性能要求

- 页面首次采集应在可接受时间内完成。
- 连续 DOM 变化不应导致高频全量上传。
- 必须支持节流、去抖和增量上报策略。

## 二、指令执行

### 支持的基础指令

- `click`：点击元素
- `input`：向元素输入内容
- `scroll`：滚动页面或指定区域
- `createTab`：新建标签页
- `closeTab`：关闭标签页
- `switchTab`：切换标签页
- `removeElement`：删除 DOM 元素
- `setAttribute`：修改元素属性
- `appendElement`：新增 DOM 元素
- `observeEvent`：监听元素事件

### 建议暂不作为第一期默认能力

- 任意字符串形式的 JS 执行
- 大规模批处理指令事务
- 复杂的跨域 iframe 深度控制

### 指令执行要求

- 每条指令必须有唯一 `commandId`。
- 指令必须支持超时控制。
- 指令执行前必须校验参数。
- 指令执行后必须返回成功或失败结果。
- 对于可能影响页面稳定性的操作，应提供审计日志。
- 页面元素定位默认使用 XPath 语法。
- `selector` 仅作为兼容或调试辅助字段，不作为主定位方案。

## 三、配置页： Chrome Extension Popup 页面

### 必备配置项

- 服务端基础地址
- 服务器联通测试按钮
- 插件启动开关

### 配置页功能
- 当前插件版本显示
- 查看接口文档入口

## 四、错误处理

需要覆盖以下异常场景：

- 服务端不可达
- 指令格式非法
- 元素不存在
- 当前页面已跳转
- 指令超时
- 插件权限不足
- 目标标签页不存在

## 五、日志与审计

建议插件至少记录以下日志：

- 连接建立与断开
- DOM 上传开始与结束
- 指令接收与执行结果
- 错误码与错误信息
- 配置变更记录
