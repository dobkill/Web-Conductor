# 项目概述

## 项目目标

开发一个 Chrome 插件，使其能够：

1. 采集当前页面的 DOM 结构并发送到服务端。
2. 接收服务端下发的页面操作指令并在浏览器中执行。
3. 提供 Chrome Extension Popup 页面用于设置服务端地址、测试服务器联通性、控制插件启动开关和查看接口说明。

## 项目定位

该插件本质上是一个“浏览器自动化执行代理”，负责在浏览器端建立受控执行环境，让服务端可以基于页面结构进行分析与调度。

## 术语说明

- 插件端：Chrome 插件自身，包含后台脚本、内容脚本与 Popup 页面。
- 服务端：负责接收 DOM、生成指令、维护会话与审计日志的后端服务。
- 指令：服务端下发给插件的一个标准化操作命令。
- 目标元素：指令执行时需要定位的页面元素。
- DOM 快照：经过裁剪和脱敏后的页面结构描述数据。

## 核心使用流程

1. 用户安装并启用插件。
2. 用户在 Popup 页面中填写服务端地址并控制插件启停。
3. 插件后台与服务端建立连接。
4. 内容脚本采集当前页面 DOM 快照并上传。
5. 服务端返回一个或多个标准化指令。
6. 插件在目标页面中执行指令，并将执行结果回传。

## 补充说明

- 当前主交互入口为 Popup 页面，不再以独立配置页作为默认交互载体。
- 当前需求包含上传 `cookie`、`token`、`authorization` 相关值的能力，必须在协议、安全和实现中单独处理。

## 非目标

以下内容不建议作为第一期目标：

- 完整复刻 Puppeteer 级别的浏览器自动化能力。
- 支持任意远程脚本字符串直接执行。
- 支持复杂跨站工作流编排平台。
- 支持所有 iframe、shadow DOM、跨域子页面的无差别控制。

## 第一性原则

- 安全优先于灵活性。
- 固定指令优先于远程脚本。
- 可审计优先于隐式行为。
- 可恢复优先于一次性成功。
