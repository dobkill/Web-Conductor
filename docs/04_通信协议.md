# 通信协议

## 一、连接方式

推荐优先使用 WebSocket 进行插件与服务端的长连接通信。

理由：

- 支持服务端主动推送指令
- 适合回传执行结果与状态变更
- 简化轮询开销

轮询可作为降级方案，但不建议作为主方案。

## 二、消息通用格式

所有消息建议统一使用 JSON：

```json
{
  "id": "msg_123",
  "type": "command",
  "timestamp": 1710000000000,
  "clientId": "browser_agent_01",
  "tabId": 123,
  "frameId": 0,
  "payload": {}
}
```

## 三、消息类型

### 1. 连接注册

```json
{
  "id": "msg_connect_001",
  "type": "connect",
  "timestamp": 1710000000000,
  "clientId": "browser_agent_01",
  "payload": {
    "pluginVersion": "0.1.0",
    "browser": "chrome",
    "manifestVersion": 3
  }
}
```

### 2. 心跳

```json
{
  "id": "msg_ping_001",
  "type": "heartbeat",
  "timestamp": 1710000005000,
  "clientId": "browser_agent_01",
  "payload": {
    "status": "alive"
  }
}
```

### 3. DOM 上传

```json
{
  "id": "msg_dom_001",
  "type": "domSnapshot",
  "timestamp": 1710000010000,
  "clientId": "browser_agent_01",
  "tabId": 123,
  "frameId": 0,
  "payload": {
    "url": "https://example.com",
    "title": "Example",
    "viewport": {
      "width": 1440,
      "height": 900
    },
    "snapshot": {},
    "sensitiveContext": {
      "cookie": "",
      "token": "",
      "authorization": ""
    }
  }
}
```

说明：

- `snapshot` 用于承载结构化 DOM 信息。
- `sensitiveContext` 用于承载高敏感上下文字段。
- 建议服务端对 `sensitiveContext` 采用单独处理和审计策略。

### 4. 指令下发

```json
{
  "id": "msg_cmd_001",
  "type": "command",
  "timestamp": 1710000020000,
  "clientId": "browser_agent_01",
  "tabId": 123,
  "frameId": 0,
  "payload": {
    "commandId": "cmd_001",
    "action": "click",
    "timeout": 5000,
    "target": {
      "xpath": "//*[@id=\"submit\"]"
    },
    "params": {}
  }
}
```

### 5. 指令结果回传

```json
{
  "id": "msg_result_001",
  "type": "commandResult",
  "timestamp": 1710000020500,
  "clientId": "browser_agent_01",
  "tabId": 123,
  "frameId": 0,
  "payload": {
    "commandId": "cmd_001",
    "success": true,
    "errorCode": null,
    "errorMessage": null,
    "result": {}
  }
}
```

## 四、指令模型

### 通用字段

- `commandId`：指令唯一标识
- `action`：指令类型
- `timeout`：超时时间
- `target`：目标元素或目标上下文
- `params`：执行参数

### 目标定位建议

建议按优先级支持以下定位字段：

- `xpath`
- `selector`
- `text`
- `index`
- `frameId`

说明：

- `xpath` 是主定位字段，页面操作默认依赖 XPath。
- `selector` 为兼容字段，仅建议作为辅助信息保留。
- 服务端应优先消费 DOM 快照中的 `xpath`。

### 建议的指令定义

#### click

```json
{
  "commandId": "cmd_click_001",
  "action": "click",
  "timeout": 5000,
  "target": {
    "xpath": "//*[@id=\"login\"]"
  },
  "params": {}
}
```

#### input

```json
{
  "commandId": "cmd_input_001",
  "action": "input",
  "timeout": 5000,
  "target": {
    "xpath": "//input[@name='username']"
  },
  "params": {
    "value": "demo"
  }
}
```

#### scroll

```json
{
  "commandId": "cmd_scroll_001",
  "action": "scroll",
  "timeout": 3000,
  "target": null,
  "params": {
    "top": 800,
    "behavior": "smooth"
  }
}
```

## 五、错误码建议

- `INVALID_COMMAND`
- `INVALID_TARGET`
- `TARGET_NOT_FOUND`
- `TAB_NOT_FOUND`
- `FRAME_NOT_FOUND`
- `PERMISSION_DENIED`
- `TIMEOUT`
- `UNKNOWN_ERROR`

## 六、重试建议

- 网络发送失败可由后台进行有限重试。
- 页面操作失败不应盲目自动重试，需根据错误码决定。
- 元素未出现时可支持短时等待后重试一次或少量次数。
